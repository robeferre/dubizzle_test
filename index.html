Question 1

Describe any ways you can think of to improve the performance of a busy
website.

It is a wide question, I can start to think at my data-center hardware and network infrastructure, 
Do they servers are fully protected against security intrusions, downtime, clustered properly, doing backups?
If yes, we can walk throw the server side, using "Django" as an example, are models fully abstracted, can we 
think on way to something faster, redesign some requirements, stressing tests and debuging to find potential gaps. 
The front-end performance is responsible for a very large part of a websiteâ€™s response time. To improve 
and debug front end performance can think on profile our pages to find culprints (Firebug), always save images 
optimized for the web, minimize as much as possible my CSS and Javascript saving a few bytes, 
Combine CSS and JavaScript, CSS sprites, server-side compression and of course the memchache solution used at exercise 3a.

Question 2

Releasing functionality as an API has been a common function of Web 2.0
websites and has fuelled the exponential growth of twitter. What functionality
do you think dubizzle could create in API form to improve the quality of its
services?

I think a "Featured" tab as we can see in eBay (http://www.ebay.com/) its a good functionality 
I really enjoy study things like purchasing trends, market trend, data analysis. I have also thought that 
dubizzle web site database must be really huge. I think an api to connect to some BI solution, like MS Bids, 
IBM Cognos and any other flavor of Business intelligence solution could be developed,
also as example we can use python to collect data at all databases and insert this information on a Data warehouse.
